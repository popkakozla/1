# Лабораторная работа №3

## Тема работы
Использование приемов работы с файловой системой NTFS. Назначение разрешений доступа к файлам и папкам.

## Цель работы
Изучение механизмов разграничения доступа в файловой системе NTFS, а также получение практических навыков по настройке прав пользователей и групп.

## Теоретические сведения
Файловая система NTFS поддерживает разграничение доступа к файлам и папкам с помощью системы разрешений. Для каждого объекта можно назначать права определенным пользователям или группам пользователей. Также NTFS поддерживает наследование разрешений и изменение владельца объекта.

Основные типы разрешений:
- Read (чтение);
- Write (запись);
- Modify (изменение);
- Full Control (полный доступ).

---

## Ход выполнения работы

В ходе выполнения лабораторной работы были созданы каталоги Temp1, Temp2 и Temp3. Для созданных объектов были исследованы параметры безопасности и разрешения доступа.

Для папок были просмотрены стандартные группы пользователей:
- Administrators;
- Users;
- SYSTEM.

Через вкладку «Безопасность» были изучены текущие разрешения доступа. Далее с помощью кнопки «Изменить» были настроены права пользователей и групп.

Для группы Administrators был назначен полный доступ к объектам файловой системы, а для группы Users были оставлены разрешения только на чтение и выполнение.

Дополнительно были исследованы механизмы наследования разрешений. Через расширенные параметры безопасности наследование было отключено, после чего система преобразовала унаследованные разрешения в явные.

Также была изучена вкладка «Владелец», содержащая информацию о текущем владельце файла или папки. Были рассмотрены особенности копирования и перемещения объектов в файловой системе NTFS и изменение разрешений при этих операциях.

---

## Вывод
В результате выполнения лабораторной работы были изучены основные механизмы защиты файловой системы NTFS, способы назначения разрешений доступа, наследование прав и управление владельцами объектов файловой системы.
